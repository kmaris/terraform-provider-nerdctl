# nerdctl Provider

Manage containers with nerdctl/containerd, locally or over ssh.

containerd has no remote-manageable API like the Docker Engine API — nerdctl
is the layer that provides networking (CNI), named volumes, restart policies,
and port publishing. This provider treats the nerdctl CLI as its contract and
shells out to it (via `ssh` for remote hosts), parsing `inspect` JSON for
reads. Resource shapes are modeled on the
[kreuzwerker/docker](https://registry.terraform.io/providers/kreuzwerker/docker/latest)
provider.

## Example Usage

```terraform
provider "nerdctl" {
  host         = "ssh://user@host.example.com:22"  # omit to run nerdctl locally
  ssh_opts     = ["-i", "~/.ssh/deploy_key"]       # extra ssh options
  sudo         = false                             # run nerdctl under `sudo -n`
  namespace    = "default"                         # containerd namespace
  nerdctl_path = "nerdctl"                         # binary path on the target host
}
```

## Requirements

- `nerdctl` and containerd on the target host, rootful or rootless.
- `iptables` installed on the host — CNI networking needs it. On remote
  hosts, the provider extends the non-interactive ssh `PATH` with the sbin
  directories where Debian-family systems keep it.
- For remote hosts: non-interactive ssh (key auth in your agent) and a
  POSIX login shell on the target.
- For rootful containerd as a non-root user: passwordless sudo and
  `sudo = true`.

## Remote hosts

ssh connections run in batch mode with a 30-second connect timeout. Entries
in `ssh_opts` are passed through to the `ssh` command ahead of those
defaults, so they take precedence (identity files, jump hosts, and option
overrides all work).

## Rootless containerd

Rootless hosts work without `sudo` but need host preparation — lingering
so containerd outlives ssh sessions, cgroup delegation for resource limits,
and a sysctl for privileged ports. See the
[Preparing a rootless containerd host](guides/rootless-host) guide.

## Schema

### Optional

- `host` (String) Remote host to run nerdctl on, as `ssh://[user@]host[:port]`. Requires non-interactive ssh (key auth). Runs nerdctl locally when unset.
- `namespace` (String) containerd namespace to operate in. Defaults to `default`.
- `nerdctl_path` (String) Path to the nerdctl binary on the target host. Defaults to `nerdctl`.
- `ssh_opts` (List of String) Extra ssh CLI options for remote hosts, e.g. `["-i", "~/.ssh/deploy_key", "-J", "bastion"]`. Same shape as the docker provider's `ssh_opts`.
- `sudo` (Boolean) Run nerdctl under `sudo -n` (requires passwordless sudo). Needed for rootful containerd when connecting as a non-root user.
